# 機能追加案: カレンダービューの追加

## 優先度評価
- 優先度: 中（4位）
- 効果: 中
- コスト: 低〜中
- 判断理由: 訪問頻度を直感的に把握しやすくなり、統計画面の価値は上がる。既存の `/stats` と日付データを活用できるため実装難度は比較的抑えやすいが、登録や保存に関する根本課題を直接解決する機能ではないため、優先順位は中位。

## 概要
いつサウナに行ったかをカレンダー形式で確認できるようにします。

## 課題
- リスト表示だけでは、月のうちどれくらいの頻度でサウナに行っているか直感的に分かりにくい。

## 解決策
- `/stats` ページまたはサイドバーにカレンダー表示モードを追加する。
- 訪問があった日にアイコンやドットを表示し、クリックするとその日の記録に飛べるようにする。

## 実装イメージ
- `react-calendar` 等のライブラリを導入するか、カスタムのカレンダーコンポーネントを作成。
- `visits` データから日付ベースでマッピングして表示。
