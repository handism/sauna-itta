# 機能追加案: 画像のリサイズ・圧縮機能の導入

## 優先度評価
- 優先度: 最優先（1位）
- 効果: 高
- コスト: 低〜中
- 判断理由: 現在の保存方式は `localStorage` に画像を Base64 で直接保存しており、容量制限で保存失敗が起きやすい。既にエラーハンドリングはあるが、ユーザー価値の高い「写真付き記録」を安定して保存できない状態は優先して解消すべき。

## 概要
画像をアップロードする際、ブラウザ側でリサイズ・圧縮してから保存するようにします。

## 課題
- `localStorage` の容量制限（通常5MB程度）により、高解像度の写真を数枚保存するだけで保存に失敗する（現在のコードでもエラーハンドリングはされているが、根本解決になっていない）。

## 解決策
- `canvas` を利用して、アップロードされた画像を最大1024px程度にリサイズし、JPEG形式で圧縮してから Base64 化する。

## 実装イメージ
- `useSaunaVisits.ts` または `VisitForm.tsx` の画像処理部分に `canvas` を使ったリサイズロジックを挿入。
- `browser-image-compression` 等の軽量ライブラリの導入も検討。
